**Issue:** Insufficient filename sanitization in STL compilation
**Risk:** Malicious users could access arbitrary files via `../../../`
**Fix:** Multi-layer protection:
  - Strict character whitelist (alphanumeric + underscore/hyphen)
  - Explicit traversal sequence rejection
  - Length limits (100 chars max)
  - Path verification (must be in CAD output dir)
**Impact:** Security hardened, vulnerability eliminated
**File:** `command_center.py` compile_scad_to_stl function

**Root Cause:** Broken symlink, improper installation, quarantine attributes
**Fix:**
  - Uninstalled broken version
  - Reinstalled via Homebrew
  - Removed quarantine attributes (`xattr -r -d com.apple.quarantine`)
**Impact:** No more security warnings, OpenSCAD execution verified
